# COMMON ISSUES

## Common Student Errors
1. **Incorrect Input**: Ensure that any inputs are correctly formatted.
2. **File Not Found**: Make sure all file paths are accurate and exist in the project.
3. **Syntax Errors**: Check your code for missing semicolons, mismatched brackets, etc.

## Debugging Tips
- **Use Console Logs**: Insert `console.log()` statements in your JavaScript code to see output and debug.
- **Browser Developer Tools**: Make use of Developer Tools in your browser (F12) to inspect elements and check the console for errors.

## CSS Alignment Problems
- **Use Flexbox**: Utilize Flexbox for aligning elements. Ensure you set `display: flex;` on the parent element. 
- **Margin and Padding Adjustments**: Adjust `margin` or `padding` to help align elements properly.

## JavaScript Errors
- **Null or Undefined Reference**: Always check if the variable or object exists before calling its properties or methods.
- **Event Listeners**: Ensure event listeners are correctly attached after the DOM elements have loaded.

## Bootstrap Compatibility Issues
- **Including Bootstrap Correctly**: Make sure to include Bootstrap's CSS and JS files correctly in your HTML. Check for version compatibility. 
- **Custom CSS Overriding Bootstrap**: Check your custom CSS to see if it's overriding Bootstrap styles. Use browser inspection tools to view which styles are being applied.

## Conclusion
Consult this troubleshooting guide whenever you encounter issues. Seeking assistance from peers or instructors is also encouraged if problems persist.
